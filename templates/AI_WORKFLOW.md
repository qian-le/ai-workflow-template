# AI_WORKFLOW — Codex-Architect / Executor Workflow Guide

> This document describes the complete collaboration workflow between **Codex (Architect)** and **Claude Code or other coding agent (Executor)**.
>
> The workflow enforces clear boundaries: the Architect plans and reviews; the Executor only acts within the task assignment.

---

## Workflow Name

- Codex-Architect / Executor Enhanced Workflow

## Role Assignment

### Codex (Architect)

- Plans, decomposes tasks, defines boundaries, assesses risk, reviews, and suggests rework.
- Writes: `AI_TASK.md`, `AI_REVIEW.md`
- Never executes code changes directly.

### Claude Code or Other Coding Agent (Executor)

- Executes modifications within clearly defined boundaries.
- Runs tests, records logs, prepares handoff documentation.
- Writes: `AI_HANDOFF.md`, `AI_CHANGELOG.md`, `AI_RUN_LOG.md`
- Never expands scope, refactors broadly, or makes architectural decisions.

## File Descriptions

| File | Author | Purpose |
|------|--------|---------|
| `AI_TASK.md` | Architect | Task assignment — the single source of truth for what the Executor must do. |
| `AI_HANDOFF.md` | Executor | Post-execution handoff — what was done, what remains, what needs input. |
| `AI_CHANGELOG.md` | Executor | Per-round change log — files changed, purpose, scope compliance. |
| `AI_RUN_LOG.md` | Executor | Command log — commands run, output, errors, conclusions. |
| `AI_REVIEW.md` | Architect | Review conclusion — pass/fix/rework, findings, next-round instructions. |
| `CODEX_REVIEW_CONTEXT.md` | Script | Aggregated context generated by `collect_context.ps1` for Architect review. |

## Standard Workflow

1. **Architect** generates `AI_TASK.md` based on current requirements.
2. **Executor** follows only what `AI_TASK.md` states — nothing more.
3. **Executor** updates `AI_HANDOFF.md`, `AI_CHANGELOG.md`, `AI_RUN_LOG.md`.
4. Run `.\scripts\collect_context.ps1` in the project root.
5. Hand `CODEX_REVIEW_CONTEXT.md` to the Architect for review.
6. **Architect** produces `AI_REVIEW.md` with a verdict and next-round tasks if needed.
7. If rework is needed, **Architect** generates a new `AI_TASK.md` for the next round.

## Installation

### PowerShell Script

```powershell
powershell -ExecutionPolicy Bypass -File <TEMPLATE_DIR>\scripts\install_ai_workflow.ps1
```

Replace `<TEMPLATE_DIR>` with the path where you cloned or downloaded this template.

### Natural Language (ask Codex to install)

If the template is not yet installed in your project and you are using Codex, you can say:

```text
Please install <TEMPLATE_DIR> into the current project. Do not overwrite existing files with the same name.
```

Or with more detail:

```text
Please install the workflow from <TEMPLATE_DIR> into the current project root. Keep existing files; if a file with the same name already exists, skip it and tell me "already exists, not overwritten."
```

## Prompt for Codex (Architect)

> Read `AI_TASK.md`, `AI_HANDOFF.md`, `AI_CHANGELOG.md`, `AI_RUN_LOG.md`, `AI_REVIEW.md` (if present), and the `git diff` summary. Focus on:
>
> - Was the task objective achieved?
> - Were there any scope violations?
> - Are there obvious bugs or regression risks?
> - Is testing sufficient?
> - Should a next-round fix-up task be generated?

## Prompt for the Executor (Claude Code or Other Coding Agent)

> Execute only the tasks explicitly stated in `AI_TASK.md`. Do not expand scope. Do not refactor broadly. Do not modify `AI_TASK.md`. If you encounter uncertainty, missing information, or items beyond your authorized scope, stop at the safe boundary and write the issue to `AI_HANDOFF.md`.

## How to Run collect_context.ps1

From the project root:

```powershell
.\scripts\collect_context.ps1
```

Or from the `scripts` directory:

```powershell
.\collect_context.ps1
```

This generates `CODEX_REVIEW_CONTEXT.md` in the project root.

## How to Hand Off to Codex for Review

Provide `CODEX_REVIEW_CONTEXT.md` as the review input and explicitly request:

- Review against `AI_TASK.md` boundaries.
- List issues first, then give the verdict.
- State clearly: Pass / Needs Minor Fixes / Needs Rework.

## Important Notes

- The Executor can only execute explicitly stated tasks.
- Record uncertainties — do not guess.
- Do not skip handoff or log files.
- Do not turn this workflow into an auto-commit or auto-publish pipeline.

## FAQ

### 1. Why must `AI_TASK.md` be generated first?

Because this workflow depends on clear boundaries. Without a well-defined task, execution is likely to go out of scope.

### 2. Is `AI_REVIEW.md` required every round?

Not mandatory, but strongly recommended whenever entering review or rework phases.

### 3. What if `git` is not available?

The workflow still works. `collect_context.ps1` will note that no `git` information was detected.

### 4. Can I use an Executor other than Claude Code?

Yes. The workflow is designed to be Executor-agnostic. Any coding agent that can read `AI_TASK.md` and update the handoff/log files can serve as the Executor.
